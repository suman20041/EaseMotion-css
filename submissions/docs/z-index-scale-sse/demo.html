The Proposed Standardized z-index Scale

Every component on this page uses a named token like var(--em-z-modal) instead of an arbitrary number. Click the buttons below to see how each layer stacks predictably above the layers below it.

Token Value Intended Use
--em-z-hide -1 Visually hidden, behind everything
--em-z-base 0 Default flow layer
--em-z-raised 10 Cards, raised surfaces
--em-z-dropdown 100 Select menus, dropdowns
--em-z-sticky 200 Sticky headers, sticky nav
--em-z-fixed 300 Fixed navbars, side rails
--em-z-overlay 400 Backdrops / scrims
--em-z-drawer 500 Side drawers, off-canvas
--em-z-modal 1000 Modal dialogs
--em-z-popover 1025 Popovers anchored to triggers
--em-z-tooltip 1050 Tooltips
--em-z-toast 1100 Toasts / snackbars
--em-z-top 9999 Absolute top — debug overlays

Why a Standardized Scale?

  • Prevents arbitrary z-index: 9999 hacks across components.
  • Guarantees predictable stacking order across the framework.
  • Makes new components instantly compatible with existing ones.
  • Easy for contributors to pick the right layer by name.

Adoption Path

The :root block in style.css is ready to be copy-pasted into core/variables.css. Components in components/ can then progressively migrate from raw numbers to these named tokens.
