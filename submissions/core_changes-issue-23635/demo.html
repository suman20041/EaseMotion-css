Marquee Reduced Motion Fix

When prefers-reduced-motion: reduce is enabled (OS-level accessibility setting), the marquee animation stops and content displays as a static wrapped layout instead of overflowing.

Demo

The marquee below scrolls infinitely when motion is allowed. Enable "Reduce Motion" in your OS settings to see it switch to a static wrapped layout.

How to test

  • macOS: System Settings → Accessibility → Display → Reduce motion
  • Windows: Settings → Ease of Access → Display → Show animations
  • Linux: GNOME Settings → Accessibility → Seeing → Reduced animation
  • DevTools: Chrome Rendering tab → Emulate prefers-reduced-motion: reduce

What changed

  • Added flex-wrap: wrap so content wraps into rows instead of inline overflow
  • Added justify-content: center for centered static layout
  • Added gap to maintain spacing between items
  • Used !important to override the existing animation rule

Before vs After

Before

  • animation: none — stops scroll
  • Content still in inline-flex — overflows horizontally
  • Items remain off-screen or clipped
  • Poor user experience for reduced motion users

After

  • animation: none — stops scroll
  • flex-wrap: wrap — items wrap into rows
  • justify-content: center — centered static layout
  • All content visible and readable

Marquee Variants

The fix applies to all marquee variants seamlessly.
